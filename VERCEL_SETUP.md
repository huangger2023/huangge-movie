## 🚀 Vercel 域名绑定步骤（跟着做）

### 第 1 步：登录 Vercel

1. 打开浏览器，访问：https://vercel.com/login
2. 使用你的账号登录（GitHub/GitLab/Email）

### 第 2 步：进入项目设置

1. 登录后，你会看到项目列表
2. 找到你的项目（可能叫 `my-landing-page` 或类似名称）
3. **点击项目名称**进入项目详情页

### 第 3 步：打开域名设置

1. 在项目页面顶部，找到 **Settings** 标签，点击进入
2. 在左侧菜单中，找到 **Domains** 选项，点击进入

### 第 4 步：添加主域名

1. 在 Domains 页面，你会看到一个输入框
2. 在输入框中输入：`hgsdy.cn`
3. 点击 **Add** 按钮
4. Vercel 会自动检测 DNS 记录
5. 等待几秒，状态会变成 ✅ **Valid Configuration**

### 第 5 步：添加 www 子域名（可选但推荐）

1. 在同一个输入框中，再次输入：`www.hgsdy.cn`
2. 点击 **Add** 按钮
3. Vercel 会自动检测 CNAME 记录
4. 等待几秒，状态会变成 ✅ **Valid Configuration**

### 第 6 步：设置主域名（推荐）

1. 在域名列表中，找到 `hgsdy.cn`
2. 点击右侧的三个点 `...` 菜单
3. 选择 **Set as Primary Domain**（设置为主域名）
4. 这样所有访问都会自动跳转到 `hgsdy.cn`

### 第 7 步：等待 SSL 证书配置

1. Vercel 会自动为你的域名配置 **免费 SSL 证书**
2. 这个过程通常需要 **1-5 分钟**
3. 完成后，域名旁边会显示 🔒 图标

## 🔧 常见问题排查

### 问题 1：Vercel 提示 "Invalid Configuration"

**原因：** DNS 记录未生效或配置错误

**解决方案：**
1. 回到阿里云 DNS 控制台检查记录是否正确
2. 等待 5-10 分钟让 DNS 生效
3. 在 Vercel 点击 **Refresh** 按钮重新检测

### 问题 2：域名能访问但显示 404

**原因：** 域名绑定成功，但项目未部署

**解决方案：**
1. 回到 Vercel 项目首页
2. 检查 **Deployments** 标签
3. 确保最新的部署状态是 **Ready**

### 问题 3：访问域名显示 "This site can't be reached"

**原因：** DNS 解析未生效

**解决方案：**
1. 打开命令行（Windows 按 Win+R，输入 cmd）
2. 输入：`ping hgsdy.cn`
3. 如果显示 `76.76.21.21`，说明 DNS 已生效
4. 如果显示 "找不到主机"，再等待 10-20 分钟
